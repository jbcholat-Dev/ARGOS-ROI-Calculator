## Story

**As a** sales engineer conducting a 40+ minute ROI session with a client,
**I want** the application to automatically persist all data across page refreshes and tab closures,
**So that** I never lose work during a live demo due to accidental refresh, browser update, or tab closure.

## Acceptance Criteria

### AC1: Auto-Persist on Every State Change
**Given** I have one or more analyses with data entered
**When** I modify any field (input, global params, analysis name, etc.)
**Then** the entire store state is automatically saved to localStorage under key `argos-roi-data`

### AC2: Restore on Page Refresh
**Given** I have saved data in localStorage from a previous session
**When** I refresh the page (F5 or browser refresh)
**Then** all analyses, global params, active analysis, and exclusions are restored exactly as they were

### AC3: Session Recovery Modal
**Given** previous session data exists in localStorage
**When** the application loads
**Then** a modal appears with two options:
  - "Resume previous session" — restores all data, navigates to dashboard or last active view
  - "Start new session" — clears localStorage and starts fresh with default state

### AC4: Reset Button in UI
**Given** the application has data (persisted or active)
**When** I click a "Reset All Data" button (accessible from the app shell / settings area)
**Then** a confirmation modal asks "Are you sure? All analyses will be permanently deleted."
**And** if confirmed, localStorage is cleared and the app resets to default state

### AC5: Accessibility
**Given** the session recovery modal or reset confirmation modal is displayed
**When** I interact with it
**Then** it follows existing modal patterns: `aria-labelledby`, `aria-describedby`, focus trap (Tab + Shift+Tab), Escape to close

### AC6: Graceful Degradation
**Given** localStorage is unavailable (private browsing, storage full, disabled)
**When** the app loads
**Then** it works normally without persistence (no crash, no error modal)
**And** a console warning is logged (not visible to user)

## Technical Notes

### Zustand Persist Pattern
```typescript
import { persist } from 'zustand/middleware';

export const useAppStore = create<AppState>()(
  persist(
    (set, get) => ({
      // ... existing state and actions ...
      clearAllData: () => {
        set(initialState);
        // persist middleware auto-clears on state reset
      },
    }),
    {
      name: 'argos-roi-data',
      // Handle Set serialization
      storage: {
        getItem: (name) => {
          const str = localStorage.getItem(name);
          if (!str) return null;
          const parsed = JSON.parse(str);
          // Convert excludedFromGlobal array back to Set
          if (parsed?.state?.excludedFromGlobal) {
            parsed.state.excludedFromGlobal = new Set(parsed.state.excludedFromGlobal);
          }
          return parsed;
        },
        setItem: (name, value) => {
          // Convert Set to Array for JSON serialization
          const serializable = {
            ...value,
            state: {
              ...value.state,
              excludedFromGlobal: [...(value.state.excludedFromGlobal || [])],
            },
          };
          localStorage.setItem(name, JSON.stringify(serializable));
        },
        removeItem: (name) => localStorage.removeItem(name),
      },
    }
  )
);
```

### Session Detection Logic
```typescript
// In App.tsx or custom hook
const hasPersistedData = () => {
  try {
    const data = localStorage.getItem('argos-roi-data');
    if (!data) return false;
    const parsed = JSON.parse(data);
    return parsed?.state?.analyses?.length > 0;
  } catch {
    return false;
  }
};
```

### Edge Cases
- `excludedFromGlobal` is a `Set<string>` — needs custom serialization (Set → Array → Set)
- Private browsing mode: localStorage may throw — wrap in try/catch
- Store version migration: add `version: 1` to persist config for future-proofing
- `deploymentMode` and `connectionType` (Epic 6.4 state) should also persist
