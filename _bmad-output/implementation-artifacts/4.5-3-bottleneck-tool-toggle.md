## Story

**As a** sales engineer analyzing a process with bottleneck equipment,
**I want** to flag a tool as a "bottleneck" and apply a cost multiplier to the downtime,
**So that** the ROI calculation reflects the cascading downstream production impact when a bottleneck tool goes down.

## Background

In semiconductor fabs, some tools are "bottlenecks" — they have the lowest throughput in the production line. When a bottleneck tool goes down, it doesn't just stop that tool; it blocks ALL downstream production. The hourly cost of downtime for a bottleneck is significantly higher than the direct tool cost alone.

The multiplier is conversational: the consultant adjusts it live with the client. "When this tool goes down, how much of the line stops? Double? Triple?"

## Acceptance Criteria

**Given** I am on the analysis Focus Mode, in the downtime section
**When** the section renders
**Then** I see a toggle/checkbox labeled "Bottleneck tool" below the existing downtime fields
**And** it defaults to OFF (unchecked) for new analyses

### AC2: Conditional Multiplier Field
**Given** the "Bottleneck tool" toggle is OFF
**When** I view the downtime section
**Then** the multiplier field is NOT visible

**Given** the "Bottleneck tool" toggle is ON
**When** I toggle it
**Then** a multiplier field appears with label "Impact multiplier"
**And** it defaults to x2.0
**And** it increments in steps of 0.5 (x1.5, x2.0, x2.5, x3.0, x3.5, x4.0, x4.5, x5.0)

### AC3: Multiplier Applied to Downtime Cost Only
**Given** bottleneck is ON with multiplier x3.0 and downtimeCostPerHour = 10,000
**When** ROI calculates
**Then** effective downtime cost per hour = 10,000 x 3.0 = 30,000
**And** wafer defect cost is NOT affected by the multiplier
**And** the multiplier only affects the downtime component of total failure cost

### AC4: Calculation Formula Update
**Given** the bottleneck feature
**When** ROI calculates
**Then** the downtime cost formula becomes:
```
effectiveDowntimeCost = downtimeCostPerHour x bottleneckMultiplier (if ON, else x1)
downtimeCost = failedPumps x downtimeDuration x effectiveDowntimeCost
```

### AC5: Results Reflect Multiplier in Real-Time
**Given** I toggle bottleneck ON or change the multiplier value
**When** the change registers
**Then** the ResultsPanel updates immediately with the new total failure cost and ROI

### AC6: Duplicate Analysis Copies Bottleneck State
**Given** an analysis with bottleneck ON and multiplier x3.0
**When** I duplicate the analysis
**Then** the duplicate has bottleneck ON and multiplier x3.0

### AC7: Accessibility
**Given** the bottleneck toggle and multiplier field
**When** I interact via keyboard
**Then** the toggle is keyboard-accessible (Space to toggle)
**And** the multiplier field is keyboard-accessible
**And** appropriate aria-labels are set
**And** the conditional visibility change is communicated (aria-expanded or similar)

## Technical Notes

### Multiplier Field Options
**Option A — Select dropdown:**
```tsx
<select value={multiplier} onChange={...}>
  <option value={1.5}>x1.5</option>
  <option value={2.0}>x2.0</option>
  <option value={2.5}>x2.5</option>
  <option value={3.0}>x3.0</option>
  <option value={3.5}>x3.5</option>
  <option value={4.0}>x4.0</option>
  <option value={4.5}>x4.5</option>
  <option value={5.0}>x5.0</option>
</select>
```
**Option B — Number input with step:**
```tsx
<input type="number" min={1.5} max={5.0} step={0.5} />
```
Recommend **Option A (select)** — fewer validation edge cases, client can see all options at once.

### Calculation Integration (with Story 4.5.2 decoupled formula)
```typescript
// Combined with wafer defect decoupling:
const failedPumps = pumpQuantity * (failureRatePercentage / 100);
const waferDefectCost = waferDefectEventsPerYear * waferCost * effectiveWaferQty;
const effectiveMultiplier = isBottleneck ? bottleneckMultiplier : 1;
const downtimeCost = failedPumps * downtimeDuration * downtimeCostPerHour * effectiveMultiplier;
const totalFailureCost = waferDefectCost + downtimeCost;
```

### Edge Cases
- Toggle OFF → multiplier reverts to x1 in calculation (field value preserved for re-toggle)
- Multiplier field not cleared when toggling OFF (UX: user toggles back ON, value is still there)
- New analyses default to OFF + x2.0
- What-If scenario: bottleneck state should be copied in What-If duplication
