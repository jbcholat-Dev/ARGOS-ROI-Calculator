## Story

**As a** sales engineer modeling ROI for a client,
**I want** to enter a number of wafer defect events per year that is independent from the number of pump failures,
**So that** the ROI calculation reflects reality: not every pump failure causes a wafer defect.

## Background

Currently, the model applies wafer cost to EVERY pump failure:
```
costPerFailure = (waferCost x waferQuantity) + (downtimeDuration x downtimeCostPerHour)
totalFailureCost = failedPumps x costPerFailure
```

In reality, only a fraction of failures cause wafer defects. Example: 9 pumps, 3 failures/year, but only 1 wafer defect event. The current model calculates 3 x waferCost instead of 1 x waferCost.

Additionally, "wafer scrap" terminology is incorrect — not all wafer events are scraps (some wafers can be reworked). The correct neutral term is "wafer defect".

## Technical Notes

### New Calculation Formula
```typescript
// BEFORE (coupled):
const failedPumps = pumpQuantity * (failureRatePercentage / 100);
const costPerFailure = (waferCost * effectiveWaferQty) + (downtimeDuration * downtimeCostPerHour);
const totalFailureCost = failedPumps * costPerFailure;

// AFTER (decoupled):
const failedPumps = pumpQuantity * (failureRatePercentage / 100);
const waferDefectCost = waferDefectEventsPerYear * waferCost * effectiveWaferQty;
const downtimeCost = failedPumps * downtimeDuration * downtimeCostPerHour;
const totalFailureCost = waferDefectCost + downtimeCost;
```

### Important: isAnalysisCalculable Update
The current gate requires `waferCost > 0`. With decoupling, a valid analysis can have:
- `waferDefectEventsPerYear = 0` and `waferCost = 0` → downtime-only ROI (still valid)
- Need to adjust: calculable if `(waferDefectEventsPerYear > 0 && waferCost > 0) || (downtimeDuration > 0 && downtimeCostPerHour > 0)`
- At least ONE cost component must be non-zero

### Edge Cases
- `waferDefectEventsPerYear = 0` → wafer defect cost = 0, only downtime contributes
- `waferDefectEventsPerYear > failedPumps` → valid (a single failure can affect multiple wafer batches)
- Existing analyses without the field → default to 0 (backward compatible)
- Mono wafer mode: effectiveWaferQty = 1, so waferDefectCost = events x cost x 1

### Persistence Compatibility (with Story 4.5.1)
If 4.5.1 ships first: localStorage may contain analyses without `waferDefectEventsPerYear`. The persist `version` field + migration can handle this, OR simply default missing field to 0 at read time.
