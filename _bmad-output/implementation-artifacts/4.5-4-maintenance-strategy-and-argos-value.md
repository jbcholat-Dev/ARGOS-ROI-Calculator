## Background

### Two Maintenance Strategies with Different ARGOS Value

**Unplanned (Run to Fail + Opportunistic PM):**
- Pumps run until failure or are pulled based on subjective signals (technician hears noise)
- ARGOS value: **Anticipate** failures before they happen — detect early, pull pump days/weeks before catastrophic failure
- The number of overhauls stays THE SAME (pump still gets overhauled either way)
- Gain = avoided downtime cost + avoided wafer defect cost
- Client vocabulary: "run to fail" (GlobalFoundries), "PM removal" (opportunistic)

**Planned (Fixed-Interval Preventive Maintenance):**
- Pumps are removed on a fixed schedule (every 12, 18, 24 months) regardless of condition
- Intervals are often conservative (based on statistics, divided by safety factor)
- ARGOS value: **Extend interval** via data-driven condition monitoring
- Fewer overhauls per year = direct cost savings
- Annualization: overhauls/year = pumps / (interval in years) — steady-state across staggered pumps
- Optional: unplanned failures despite PM schedule (residual events)

### Key Insight from Party Mode Discussion
- No "hybrid" mode needed — Planned mode with optional residual unplanned events covers real-world hybrid scenarios
- Overhaul cost is single (no RTF vs PM distinction — marginal difference, client refurbishes most parts regardless)

### AC3: Unplanned Mode — Calculation
**Given** Unplanned mode with: 20 pumps, 15% failure rate (= 3 failures/year), 2 wafer defect events, €500K wafer cost, 6h downtime, €15K/h, detection rate 80%, overhaul cost €25K
**When** ROI calculates
**Then**:
```
failedPumps = 20 x 0.15 = 3
downtimeCost = 3 x 6 x 15,000 = €270,000
waferDefectCost = 2 x 500,000 = €1,000,000
totalCostWithoutArgos = €270,000 + €1,000,000 = €1,270,000
anticipatedEvents = 3 x 0.80 = 2.4 failures anticipated
costAvoided = (2.4/3) x €270,000 + (2.4/3) x waferDefectCost?
```
**IMPORTANT — Calculation refinement needed:**
The detection rate applies to the PROPORTION of failures anticipated. For downtime, this is straightforward. For wafer defect, the relationship is:
- ARGOS anticipates 80% of failures → 80% of downtime events avoided
- Wafer defect events are entered separately and may not correlate 1:1 with failures
- **Simplification:** Apply detection rate to BOTH downtime and wafer defect costs as a reduction factor
```
savingsFromArgos = detectionRate x totalCostWithoutArgos
netSavings = savingsFromArgos - argosServiceCost
```
**And** overhaul cost is NOT reduced (same number of overhauls, just anticipated)
**And** overhaul cost is shown as a separate line (informational, not part of gain)

```

argosIntervalMonths = 24 x 1.20 = 28.8 months

overhaulSavings = 1.67 x €25,000 = €41,750

residualSavings = (if residual > 0): detectionRate x residualDowntimeCost
totalSavings = overhaulSavings + residualSavings - argosServiceCost
```

## Technical Notes

### Component Architecture
```
InputPanel
├── MaintenanceStrategySelector (NEW)
├── EquipmentInputs (existing — always visible)
├── FailureRateInputs (existing — Unplanned only)
├── PlannedMaintenanceInputs (NEW — Planned only)
│   ├── PM Interval
│   ├── MTBF Extension Slider
│   ├── Overhaul Cost
│   └── Unplanned Despite PM (optional)
├── WaferInputs (existing — always visible, uses waferDefectEventsPerYear from 4.5.2)
├── DowntimeInputs (existing — always visible, includes bottleneck from 4.5.3)
└── DetectionRateInput (existing — Unplanned only)
```

### Detection Rate Consolidation
Currently, `detectionRate` exists both as a global param and per-analysis override. In the new model:
- **Unplanned mode**: detection rate is the core ARGOS slider (keep existing behavior)
- **Planned mode**: MTBF extension replaces detection rate as the ARGOS value lever
- Per-analysis `detectionRate` field is still used in Unplanned mode
- Global `detectionRate` remains as default for new analyses

### Annualization Formula for Planned Mode
```
overhaulsPerYear = numberOfPumps / (intervalMonths / 12)
```
This assumes steady-state: pumps are staggered, so every year you see a constant annual rate. This is the standard industrial calculation.

### Field Preservation on Strategy Switch
When user switches from Unplanned → Planned → Unplanned:
- All field values are preserved in the Analysis object
- Only the VISIBLE fields change, not the stored data
- Calculation dispatches based on `maintenanceStrategy` value

### Edge Cases
- Planned with 0 residual + 0 wafer defect → pure overhaul savings only
- Planned with residual > 0 → hybrid calculation (overhaul savings + residual downtime savings)
- Unplanned with 0 wafer defect events → downtime-only ROI
- PM interval = 1 month → 240 overhauls/year for 20 pumps (extreme but valid)
- Switching strategy mid-session: no data loss, recalculates immediately
